# RPM Build Target Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an RPM packaging target for Fedora Linux, replacing the existing `zip` Linux output and adding a dedicated `package:linux` script.

**Architecture:** Three config-only changes — swap the electron-builder Linux target from `zip` to `rpm`, add a Fedora RPM metadata block with system library dependencies, then wire up `package:linux` scripts at both the workspace and root level.

**Tech Stack:** electron-builder (RPM target), pnpm workspaces

**Spec:** `docs/superpowers/specs/2026-03-10-rpm-build-design.md`

### Task 2: Add package:linux script to src/main/package.json

**Files:**
- Modify: `src/main/package.json:14`

- [ ] **Step 1: Add the script**

In `src/main/package.json`, after the `"package:nosign"` line (line 14), add:

```json
"package:linux": "node ./prepare-dist-package.mjs && pnpm install --dir=./dist && pnpm electron-builder --config ./electron-builder.config.json --publish never",
```

The scripts section should now contain (among others):

```json
"package": "node ./prepare-dist-package.mjs && pnpm install --dir=./dist && pnpm electron-builder --config ./electron-builder.config.json --config.win.forceCodeSigning=true --config.win.sign=./sign.js --publish never",
"package:nosign": "node ./prepare-dist-package.mjs && pnpm install --dir=./dist && pnpm electron-builder --config ./electron-builder.config.json --publish never",
"package:linux": "node ./prepare-dist-package.mjs && pnpm install --dir=./dist && pnpm electron-builder --config ./electron-builder.config.json --publish never",
```

Note: `package:linux` is intentionally identical to `package:nosign` — no `prepare:win` step, no code signing flags.

- [ ] **Step 2: Commit**

```bash
git add src/main/package.json
git commit -m "feat: add package:linux script to src/main"
```

### Task 4: Verify the build produces an RPM

> **Note:** This requires a Linux environment with `rpmbuild` installed (`dnf install rpm-build`). Run this step on a Fedora machine or in the GitLab Linux CI environment, not on macOS.

- [ ] **Step 1: Run the full build pipeline**

```bash
pnpm run package:linux
```

Expected: build completes without error.

- [ ] **Step 2: Confirm RPM artifact exists in dist/**

```bash
ls dist/*.rpm
```

Expected: a file matching `vortex-*.rpm` is present.

- [ ] **Step 3: Inspect the RPM metadata**

Expected output includes:
- `Name:` vortex (or the app name)
- `Requires:` libXScrnSaver, libXtst, nss, alsa-lib, gtk3, libnotify
